Claat

Instalación mediante el ejecutable, también es posible realizarla mediante Go, lo que significa que tiene que instalar Go.
Descargamos el ejecutable y lo ubicamos en la carpeta donde planeamos guardar nuestros codelabs, el único inconveniente de esto:

Al ejecutar cualquier comando de claat debemos hacerlo llamando al ejecutable directamente, seguido del comando a llamar: claat.exe help. Esto en caso de utilizar el ejecutable.

Quisiera empezar enseñando la página que servirá nuestros codelabs, también miraremos las principales funciones basicas.

El objetivo es personalizar la página que sirve los codelabs y lograr grandes presentaciones como lo es: Google Codelabs

Preparando

Para esto nos dirigimos al repositorio de herramientas y clonamos el repositorio: googlecodelabs/tools: Codelabs management & hosting tools (github.com).

La carpeta que contiene nuestra plantilla es llamada site, accedemos a esta carpeta e instalamos las dependencias.

cd site/

npm install

Para lanzar el servidor local debemos ejecutar los comandos en una terminal con permisos de administrador, este es un inconveniente que resolveremos más adelante. Si se presenta algún problema intente identificar el problema que arroja la terminal, probablemente necesite activar la ejecución de scripts

Ahora vamos a servir nuestro sitio en modo de desarrollo.

gulp serve

Podemos ver el servidor ejecutándose de forma predeterminada en: http://localhost:8000/

Resultado

El resultado será un poco simple, tal vez te preguntarás que esto no se parece nada al codelab de ejemplo mencionado anteriormente, además no se muestran codelabs. Vamos a dejar el diseño para un nuevo codelab, por ahora vamos a ver las funciones que podemos agregar.

Plantilla de ejemplo

Copiamos el siguiente ejemplo de codelab a nuestro drive para que podamos editarlo: Example Codelab - Your first Progressive Web App - Documentos de Google

Visualizar cambios

Para visualizar como esta quedando nuestro codelab tenemos que instalar una extensión: Preview Codelab - Chrome Web Store (google.com)

Ubicados en nuestro codelab, seleccionamos la extensión y automáticamente muestra el codelab.

También puedes visualizar navegando manualmente e incluyendo al final de la siguiente dirección el id de su documento: https://codelabs-preview.appspot.com/?file_id=

Guía de funcionalidades que podemos incluir

Para incluir funcionalidades a nuestro codelab podemos revisar la siguiente guía: Codelab Formatting Guide (External) - Documentos de Google

  1. Vamos a ubicarnos en la carpeta donde queremos guardar nuestro codelab, en esa carpeta tiene que estar ubicado el programa claat.
  2. Para compilar nuestro archivo de google docs necesitamos identificar el ID del documento, este lo podemos encontrar en la dirección url, observemos la siguiente imagen:
  3. Para ejecutar la exportación vamos a ubicarnos en la carpeta que tenemos el programa claat y ejecutaremos lo siguiente : .\claat.exe export "ID obtenido en el punto 2"

Ahora tenemos nuestro codelab en una carpeta con el nombre del codelab. Vamos a servir este codelab en nuestra página principal.

Si lanzamos nuestra página principal sin ninguna bandera, esta no detectará ningún codelab, para esto necesitamos crear una carpeta que contenga los codelabs creados.

bandera para apuntar a los codelabs

En nuestra proyecto hemos creado una carpeta codelabs, la cual contendrá nuestros codelabs, ubicados en el proyecto que contiene a la carpeta de nuestros codelabs y ejecutamos:

gulp serve --codelabs-dir=codelabs
