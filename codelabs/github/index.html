Github es un servicio de repositorio en la nube, el cuál esta basado en Git, vamos a hablar un poco mas de Git.

Git

Git es un servicio de control de versiones, el cual nos permite administrar el trabajo de un gran número de desarrolladores, Git fue desarrollador por linux torvalds, el mismo desarrollador de linux.

Cuando quieres realizar un cambio, creas una copia, luego cuando quieres incorporar los cambios copias el archivo modificado al original perdiendo el historial de tu trabajo. Esto es un error, ya que no podrás regresar en el tiempo en caso de que lo necesites

Lo más recomendado seria usar un control de versiones como git, este realizar los cambios e incorporalos al archivo original, todo esto conservando el historial de tu trabajo, en caso de que sea necesario podrás regresar a archivos del pasado y recuperarlos.

En esta sección mostraré 2 formas de instalar Git en Windows, la primera es a través del ejecutable y la segunda es por medio de los archivos.

Ejecutable

Podemos dirigirnos directamente a la página de Git git-scm.com, la página nos detectará el sistema operativo que estemos utilizando y nos dará la opción de descarga adecuada para nuestro sistema operativo. Si quieres saber más sobre Git en windows, te dejamos la paginá recomendada por Git.

Archivos

La instalación mediante archivos la realizo por motivos personales, si no tienes ninguna necesidad de realizar esto, la mejor opción es por medio del ejecutable.

En esta explicación nos vamos a extender un poco, ya que es la que actualmente estoy utilizando, ademas de que puede ser un poco confusa.

Para descargar los archivos tenemos que navegar hasta el código fuente de Git para windows, una vez hay podemos dirigirnos a ultimo lanzamiento realizado. Al final de la pagina podemos ver diferentes archivos de diferente tipo ya sea comprimidos o ejecutables, vamos a seleccionar un archivos comprimido. Identificaremos el archivo que necesitemos en base al nombre del archivo. Git-2.32.0-64-bit.tar.bz2, como podemos ver las especificaciones son separadas por un guion, nombre-version-arquitectura-tipoArchivo.tipoArchivo.

Ruda de los archivos

Tendremos que acceder a esa opción, a pesar de que nos arroje la descarga simplemente la cancelamos, lo que nos importa es acceder a la página de la post descarga de Windows, también podemos acceder navegando en download/win.
  • La lupa enfoca la opción para acceder al código fuente.

Identificación archivo de descarga

Instalar los archivos

Una vez tengamos el archivo descargado, descomprimimos estos archivos en la ruta donde normalmente instalamos los archivos.

Generalmente realiza esta instalación por que utilizo esta forma para actualizar git en mi entorno de desarrollo con Laragon, por lo que mi ruta de instalación seria ‘letra disco de instalación':\laragon\bin . Si no necesitas hacer estas movidas raras, lo mejor es realizar la instalación por medio del ejecutable para menos complicaciones.

Instalar los archivos

Al realizar la instalación mediante este método, nuestro sistema operativo no sabra donde encontrar git, por lo que vamos agregar las variables de entorno de manera manual.

Voy a suponer que tienes el conocimiento básico de manejar Windows.

  • En el menu de inicio buscamos ‘Editar las variables de entorno del sistema', elegimos la aplicación que cuente con ese nombre, generalmente suele ser la primera. Se abrirá una ventana de propiedades del sistema, en esta buscamos entre las ultimas opciones ‘Variables de entorno' en la cual vamos a acceder.
  • Si realizaste correctamente los pasos deberías encontrarte con una ventana igual a la siguiente imagen.
  • En la ventana encontraras dos apartados parecidos, en el primero son para las variables del usuario y el segundo es para las del sistema. Si estás trabajando en un computador personal, para no tener problemas te recomiendo agregar las variables de entorno en el sistema.
  • Seleccionar la opción de ‘PATH' y seleccionas la opción de editar como lo muestra la imagen anterior.
  • Muy bien, ahora en la nueva ventana buscamos la única opción que diga ‘Examinar', buscamos la ruta donde instalamos nuestro Git y en Git buscamos nuevamente el directorio bin, el directorio bin contiene los scripts que nuestros sistema necesitara.

¡Felicidades!. Ahora tienes Git instalado en Windows, también identificasté donde encontrar el código fuente de Git ademas de diferentes instaladores.
