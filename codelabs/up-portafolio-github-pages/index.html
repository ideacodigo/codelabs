Portafolio

El portafolio debe estar construido unicamente en html, css y js.

Utilizare el portafolio jaume, el cual lo puedes encontrar en la pagina leonidasesteban.com.

Iniciando el repositorio

   Git init

Confirmando archivos

   Git add .

Verificando estado

   Git status

Confirmando los archivos

   Git commit -m 'first commit portafolio'

Cuenta en Github

Vamos a obviar esta sección ya que puedes crear una cuenta en Github muy fácilmente, te puedes apoyar de tutoriales en internet.

La cuenta puede ser gratuita.

Igualmente para ingresar, ingrese tu usuario y contraseña, solo confirmas tu inicio de sección en el correo y ya lo tienes.

Formulario de creación

Hay varias rutas u opciones en gitHub para acceder al formulario parar crear un nuevo repositorio, miremos algunas.

Menu de opciones

Una vez hallamos ingresado, nos ubicamos en el menu superior y en este menu buscamos un sud-menu desplegable en el cual podemos encontrar la opción de new repository. Miremos un recorte para identificarlo mejor

Barra de dirección

Nos ubicamos en la barra de direcciones del navegador y ya habiendo ingresando en Github, anexamos a la dirección de github.com /new

LLenado el formulario de creación

Nombre

Muy bien, hora en nuestro formulario de creación del repositorio vamos a ingresar un nombre, prueba ingresar cualquier nombre, github te advertirá cuando el nombre del repositorio no este disponible o tenga algún carácter inadecuado.

Descripción

La descripción como lo menciona es opcional, siempre es recomendable agregar una y más aún si piensas colaborar con otras personas.

¿Publico o privado?

Seleccionamos un repositorio publico para que tu portafolio lu pueda ver cualquier persona interesada en tu trabajo. Puedes crear repositorios privados, aunque estos cuenten con algunas opciones limitadas para una cuenta gratuita.

Finalizando

Muy bien, ya lo tienes. Una vez ya hayas llenado las opciones mencionadas anteriormente estas listo para finalizar la creación del formulario.

Probablemente hayas notado que no mencionamos otras opciones como la licencia, el Readme o el archivo .gitignore, no te preocupes, miraremos estas opciones en otra ocasión, pero para seguir al pie con esta guía, no las vamos a marcar ninguna de las anteriores opciones.

Una vez hayas confirmado la creación del repositorio, Git mostrar una configuración rápida para conectar tur repositorio local con el repositorio en github, el cuál ya hemos creado.

Opciones de configuración.

La configuración rápida que nos es mostrada no ofrece tres opciones, vamos a revisarlas rápidamente, nos extenderemos un poco mas en la cuál vallamos a utilizar.

Configuración rápida.

En la configuración rápida nos es mostrada dos opciones:

  1. Esta nos indica que realizamos la conexión con la aplicación de github para escritorios, en caso de que la tengamos instalada.
  2. Nos es facilitado las direcciónes, ya sea https o ssh, con esto asumen que ya tenemos conocimientos básicos y unicamente con la dirección ya tenemos suficiente.

Ya empezando nos es recomendado algunas cosas, entre ellas crear los archivos que no hemos seleccionado en el formulario de creación. Pero en este caso la hacemos adrede para agregarlos posteriormente, mas adelante realizaremos un ejercicio agregándolos por defecto.

Creando un repositorio por medio de linea de comandos.

Esto se refiere a iniciar el repositorio localmente y conectarlo mediate linea de comandos. No utilizaremos este paso ya que hemos preparado el proyecto anteriormente en la sección Preparando el portafolio.

echo "# diego0at" >> README.md
Git init
Git add README.md
Git commit -m "first commit"
Git branch -M main
Git remote add origin https://github.com/diego0at/diego0at.git
Git push -u origin main

Enviar a un repositorio existente desde la linea de comando.

Como ya tenemos listo nuestro repositorio localmente, lo único que nos falta es enviar los archivos, por lo que vamos a utilizar este paso para nuestro proyecto.

Este paso es que utilizaremos para nuestro proyecto de ejemplo.

Bien, ahora antes ingresar los siguientes comandos, no olvides ubicarte en el proyecto desde la terminal, verifica que tus archivos estén listos con Git status, esto no debe mostrar archivos pendientes.

git remote add origin https://github.com/diego0at/diego0at.git
git branch -M main
git push -u origin main

Ahora puedes recargar la página de la configuración recomendada por github, y ahora en vez de las recomendaciones mostrara los archivos de tu proyecto.
